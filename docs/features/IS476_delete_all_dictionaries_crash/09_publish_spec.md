# Publish Spec — IS476

## Обновлённые файлы спек

### docs/features-spec/dictionary-appbar.md [~]

Зафиксировано текущее состояние:

- В разделе «Состояние (State)» уточнено, что `currentDict: DictUiEntity?` корректно представляет «нет словаря» (`null` — валидное состояние).
- В разделе «Иконка текущего словаря» добавлен случай `currentDict == null`: иконка не отображается, виджет не рендерится хост-экраном.
- Полностью переписан подраздел «Текущий словарь» под `flowCurrentDict(): Flow<DictUiEntity?>`. Контракт эмиссий описан тремя случаями: непустой список с существующим id, непустой список с orphaned id (fallback на первый), пустой список (`null`).
- Добавлен новый раздел «Пустой список словарей»: поведение виджета при `currentDict == null && availableDictList.isEmpty()` — хост скрывает AppBar; при гонке во время удаления виджет корректно отрабатывает `null` без креша.
- В разделе «Dropdown-меню» уточнено, что при пустом списке пункт «Создать словарь» остаётся доступен, а пункты со словарями отсутствуют.
- В таблице Messages у `CurrentDict` явно прописан тип параметра `DictUiEntity?`.

### docs/features-spec/dictionary-list.md [~]

Зафиксировано текущее состояние:

- В разделе «Удаление» расширены шаги 3-4: при удалении текущего словаря useCase либо переключает pref на первый оставшийся, либо очищает `CURRENT_DICTIONARY_ID_LONG` (`setLong(_, null)`), если оставшихся нет. Все активные подписки на текущий словарь эмитят `null`, без исключений.
- Добавлен новый раздел «Инварианты pref'а текущего словаря». Описаны три состояния pref'а (`null`, существующий id, orphaned id) и обязательное поведение трёх читателей: `DictionaryAppBarUseCase.flowCurrentDict()`, `DictionaryTabUseCase.flowCurrentDict()/getCurrentDict()`, `QuizChatUseCase.getCurrentDictionaryId()`. Зафиксирован общий инвариант «удаление любого словаря не приводит к необработанному исключению ни в одной активной подписке».
- Раздел «Навигация назад» расширен в «Навигация назад и user-journey пустого списка». Conditional `RequestBack` показан как часть продуктового user-journey: удалил все словари → `ExitApp` → `activity.finish()` → следующий запуск → Splash видит пустую БД → `openDictionarySetup()` → форма создания первого словаря (`DICTIONARY_SETUP`). Добавлена кросс-ссылка на `dictionary-create.md`.
- Добавлен новый раздел «Системное ограничение: QuizChat»: контракт `getCurrentDictionaryId(): Long?` и поведение `QuizGameImpl` при `null` (warning + пустой quiz-list). Объяснено, почему в норме пользователь не попадает в квиз-чат без словарей, и зачем нужна защита.
- В разделе «Зависимости» у `DictionaryUseCase` добавлена ответственность «чистка orphaned pref при удалении последнего словаря».
