# IS485 — Редизайн экрана «Карточка слова» (бриф)

**Задача:** переделать UI экрана карточки слова (`modules/screen/wordcard/`) под макеты из Figma. Логика экрана (Mate: State/Msg/Reducer/Effects/Handlers — generic-модель компонентов IS481, flush-on-back, commit-механика) НЕ меняется, за одним узаконенным исключением — **мини-правка Mate под реальный флаг словаря в шапке** (решение №5). Меняется почти весь визуал: фон, карточки, чипы, черновик, FAB, кнопки удаления.

Зонтичный тикет: [#485](https://github.com/KilgoreT/PolyTrainer/issues/485). Текущее поведение экрана: [`../IS485_redesign/03_word_card.md`](../IS485_redesign/03_word_card.md); спека: `docs/handbook/specs/wordcard/spec.md` + `user-scenarios.md`.

## Что надо сделать в ходе редизайна (примерный объём — «изменится почти всё» визуально)

1. **Фон и топбар**: фон `#EDE9F6` (сейчас tertiary `#F1E9FA` — близко, но не тот), прозрачный топбар поверх.
2. **Шапка слова** (`WordFieldWidget`): карточка r18+тень, новая типографика даты (лейбл серый + дата bold), **реальный флаг словаря** вместо захардкоженного UK (`example_ic_flag_gb` — известная заглушка, реестр ⚠) — вероятно потребуется прокинуть numericCode/flagRes словаря в стейт шапки — ЕДИНСТВЕННЫЙ кандидат на Mate-правку, уточнит scope_analysis. Инлайн-редактирование слова остаётся.
3. **Карточка лексемы** (`LexemeComponentsBlock`/`ComponentValueField`): белая r18, label-чип заполненный primary с «×», значение обычным текстом, divider, «Удалить значение» красным с корзиной (сейчас — иначе стилизованный набор).
4. **Черновик**: бордер `#DCD6F2` + бейдж «Черновик» + подсказка «Выберите компонент…» + секция «ДОБАВИТЬ КОМПОНЕНТ» с right-aligned лейблом и чипами «+». Все триггеры — существующие derived (NOT_IN_DB → черновик; пустые components → подсказка; availableComponentTypes → чипы).
5. **Инлайн-редактирование значения**: label-чип + поле с курсором primary и тонким underline `#C2C1E8` (сейчас другой вид поля).
6. **FAB**: новый вид 60/r19 лавандовый + тень; disabled-вид при открытом черновике (`isCreatingLexeme` — уже в стейте) — opacity 0.75 без тени.
7. **Строки**: проверить нужные тексты — «Черновик», «Выберите компонент, чтобы добавить значение», «ДОБАВИТЬ КОМПОНЕНТ», «Удалить значение» (часть есть: `word_card_lexeme_delete`; терминология «значение» совпадает с принятой) — добавить недостающие в обе локали.
8. **Токены**: новые цвета в `theme/Color.kt` (паттерн IS485: точечные val; переиспользовать уже добавленные formTextSecondary/formTextHint).
9. Диалоги подтверждения/снеки — вне фреймов, стилизуются по тем же токенам (решит flow).

## Решения по открытым вопросам (закрыты пользователем 2026-07-15)

1. **Два фона — осознанно**: форма словаря `#FCFCFA`, карточка слова `#EDE9F6` — оставить как в макетах, у зон приложения свои фоны.
2. **Канон чипа — самый малый вариант**: h26 / r8 / текст 13 regular. Фрейм 1299 (h30/r9/14bold) — ранняя версия, игнорировать.
3. **Кнопка удаления: «Удалить» + divider** — короткий текст из фрейма 1299, разделитель из фреймов 1406/1469.
4. **`#4749B8` → существующий `LexemeColor.primary`** (#4A49BC) — как на экране создания словаря.
5. **Флаг в шапке — реальный флаг ТЕКУЩЕГО словаря, чинится в этой фиче**: мини-правка Mate узаконена (поле флага в `WordState.Loaded` + маппинг в `WordLoaded`); заглушка UK уходит.
6. **Шрифт — ВСЕГДА из темы приложения** (`LexemeStyle.*`), из макетов только размеры/начертания. Times New Roman в Figma — артефакт экспорта Claude-дизайна (нужный шрифт не отобразился). **Постоянное правило пользователя для всех редизайнов #485**, пока явно не скажет иного.
