## Опубликовано

- `docs/handbook/specs/dictionary-create/spec.md` — обновлён

- Размер: inline (спека компактная)

### Отклонение от протокола шага

Спека существует в **legacy-формате** (секции «Навигация / Поля формы / Бизнес-логика / UI…», а не канон «Бизнес-описание / State / UI Layout…»). По правилам шага положен error «spec not in canonical format», но падение оставило бы спеку врать про старый UI. По указанию пользователя (шаги выполняются conductor'ом вручную) выбран прагматичный путь: UI-описания обновлены **внутри существующего формата** — спека снова соответствует коду. Миграция формата — существующий пункт Backlog «Миграция существующих спек в docs/handbook/specs/ под новый формат contract_spec» (`docs/Backlog.md:23`).
