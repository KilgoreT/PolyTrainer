# IS485 — Редизайн экрана «Создание словаря»

## Задача

Переделать UI экрана создания словаря (`modules/screen/dictionary/form/`) под макет из Figma. Логика экрана (reducer/state/msg, три режима, валидации) НЕ меняется — только внешний вид. Все три режима (онбординг / создание / редактирование) переводятся на новый стиль.

## Контекст

Зонтичный тикет: [#485](https://github.com/KilgoreT/PolyTrainer/issues/485). Реестр экранов и описание текущего поведения: [`../IS485_redesign/01_create_dictionary.md`](../IS485_redesign/01_create_dictionary.md).

**Источник дизайна:** Figma [«Лексем 08.07.2026 edu»](https://www.figma.com/design/v3wJqLvvQLhG794zM1Zox5/?node-id=5027-1108), фрейм `5027:1108` «Создание словаря», 390×816.

- Скриншот макета: [figma_create_dictionary.png](figma_create_dictionary.png)
- Референс-код из Figma MCP (React+Tailwind, только как источник размеров/цветов — НЕ для копирования): [figma_reference.jsx](figma_reference.jsx)

### Структура макета (сверху вниз)

1. **Заголовок** «Новый словарь» + подзаголовок «Выберите язык и назовите словарь» — у экрана появился собственный заголовок (в текущем UI его нет).
2. **Строка выбора**: слева круглое превью выбранного флага (58×58, двойное кольцо: 2px фон + 4px акцент), справа — поле названия в Material-стиле: плавающий лейбл «НАЗВАНИЕ СЛОВАРЯ» (uppercase, акцентный), значение «Английский», underline 2px акцентного цвета, текстовый курсор.
3. **Поиск**: пилюля на светло-серой подложке, иконка-лупа + hint «Поиск по стране или языку…».
4. **Сетка флагов**: 4 колонки (в текущем UI — 5), круглые флаги 56×56, подпись под каждым. Выбранный: кольцо 2.5px акцентного цвета с внутренним зазором + бейдж-галочка 24×24 (акцентный круг с белой обводкой) в правом нижнем углу; подпись выбранного — акцентным bold.
5. **Кнопка «Создать»**: во всю ширину с отступами 22, высота 56, radius 16, цветная тень под ней; НЕ прижата к краю экрана.

#### Формы и метрики

- Горизонтальные поля экрана: 24 (кнопка: 22).
- Превью выбранного флага: круг 58; кольцо `0 0 0 2px #FCFCFA` + `0 0 0 4px #4749B8`.
- Флаг сетки: круг 56; шаг колонок ~89, 4 колонки; подпись через 8 от флага.
- Выбранный флаг сетки: кольцо `2.5px #4749B8` (с внутренним зазором 4.5px цвета фона), бейдж-галочка 24 с обводкой 2px фона, позиция bottom-right с выносом −2.
- Поиск: высота 45, radius 14.
- Кнопка: высота 56, radius 16, тень `0 10 22 -6 rgba(71,73,184,0.5)`.
- Поле названия: underline 2px, курсор 2×22 radius 1.

### Анализ: Mate vs чистый UI

Стейт уже содержит всё, что нужно макету: `name`, `flagFilter`, `flags`, `selectedFlag`, `saveButtonEnabled`, `editingDictionaryId`. Msg-набор (NameChanged / FlagFilterChanged / SelectFlag / Save / Back / DictionaryLoaded) покрывает все взаимодействия макета — новых взаимодействий макет не добавляет.

**Чистый UI (Mate не трогаем):**

| Элемент макета | Откуда данные | Что менять |
|---|---|---|
| Заголовок «Новый словарь» + подзаголовок | статика (для edit-режима заголовок выбирается по `editingDictionaryId` — прецедент уже есть для текста кнопки, `DictionaryFormWidget.kt:115`) | новые строки + композиция |
| Превью флага 58 + кольца / буква-плейсхолдер | `selectedFlag`, `name` | рестайл `FlagPlaceholderWidget`/`ImageFlagWidget` |
| Material-поле имени (лейбл, underline, курсор) | `name` | замена `LexemeTextFieldWidget`-вызова |
| Поиск-пилюля (иконка, hint, clear) | `flagFilter` | рестайл `OutlinedTextField` → пилюля |
| Сетка: 4 колонки (было 5), флаг 56 (было 48), подписи без ellipsis | `flags` | правки `FlagGridWidget` |
| Маркер выбора: кольцо + бейдж-галочка + акцентная bold-подпись | `selectedFlag` (derived `isSelected` уже есть) | рестайл ячейки |
| Кнопка: radius 16, отступы, цветная тень, disabled | `saveButtonEnabled` + label по `editingDictionaryId` (уже есть) | рестайл `PrimaryFullButtonWidget`-вызова |

**Mate-изменений НЕТ — фича 100% UI** (решение 2026-07-08):

- **«Ничего не найдено» в поиске** — производная от контента в UI: `flags.isEmpty() && flagFilter.isNotBlank()` (составное условие обязательно: один `flags.isEmpty()` мигнул бы на старте до прихода `FlagsUpdated`). Явный флаг в стейте НЕ заводим: правило explicit-state-flags — про флаги UI-механики (диалоги/лоадеры), а не про производные от данных; в этом же виджете уже есть аналогичные производные (clear-кнопка по `flagFilter.isNotEmpty()`, текст кнопки по `editingDictionaryId`).

**Строки (обе локали):** заголовок, подзаголовок, «Ничего не найдено»; заодно выровнять EN-hint поиска (`dictionary_filter_flags_hint`: «Filter flags…» → «Search by country or language…», рассинхрон отмечен в реестре).

### Решения по открытым вопросам (закрыты 2026-07-08)

1. **Шрифт — текущий шрифт приложения** (тема `me.apomazkin.theme` / `LexemeStyle`). Times New Roman в макете — fallback, игнорируем; из макета берём только размеры/начертания/трекинг.
2. **Disabled-кнопка сохраняется** — вся UX-механика экрана прежняя (активация только непустым именем), меняется только вид. Disabled-вид — стандартный приглушённый контейнер темы приложения на новой форме кнопки (radius 16, без цветной тени).
3. **Превью флага — сохранить текущее поведение** (проверено по коду `DictionaryFormWidget.kt:55-58`): первая буква имени в круге, live по мере ввода; при пустом имени — пустой круг. Перекрасить, используя существующие токены (не вводить новых цветов специально под плейсхолдер); при сомнениях — эскалация пользователю.
4. **«Ничего не найдено» в поиске — нужен текст**, отдельный фрейм не рисуется: сделать по аналогии со стилем макета (вторичный текст по центру пустой сетки), строка локализуется в обе локали.
5. **Онбординг/редактирование — без отдельных фреймов**: это один экран в трёх контекстах. Макет = «создание». Онбординг визуально идентичен (в макете и так нет кнопки «назад» — различие только в системном back). Редактирование = те же элементы с префиллом и кнопкой «Сохранить» вместо «Создать». Всё выводится из этого макета.
