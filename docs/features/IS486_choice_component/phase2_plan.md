# IS486 — Фаза 2: карточка слова (ТДД)

Скоуп — [plan.md](plan.md) фаза 2. Девайс-чек-лист объединён с финалом фазы 1 (одна переустановка).

## Работы

1. **Data-слой, value-триггеры каскада**: `deleteComponentValue` / `updateComponentValue` запускают каскад-модуль (`ValueRemoved`; для смены CHOICE-значения — план по старому значению минус само значение). Каскадные потомки — soft delete; прямое пользовательское удаление значения — существующий путь.
2. **Деградация лексемы в черновик — ОТЛОЖЕНА в фазу 3** (решение 2026-07-19, вопрос 1 → вариант 2): представление пустого черновика — UI-вопрос без дизайна; старое поведение (удаление лексемы + undo) живёт до фазы 3.
3. **Правило участия в чипах** (reducer, явные поля state): для каждой лексемы addable-список = типы с активной целью (черновик — только ядра; зависимые по мере активации; CHOICE-опции активируют value-зависимых). Полный full-state assert (R-TR-003) в новых тестах.
4. **CHOICE-пикер значения**: typed-контент `ComponentValueState` (Text | Choice); commit → `ChoiceValues(optionId)`; опции CHOICE-типов доставляются в state (UseCase отдаёт опции вместе с типами); display опции = label ?: ресурс по ключу (6 строк EN/RU + маппер). **Вид пикера (решение 2026-07-19, вопрос 2 → вариант 3): диалог с радио-рядами в стиле IS485** (визуальный язык `ComponentRadioRow`, локальный виджет wordcard).
5. **Снятие времянки**: фильтр CHOICE из `WordCardUseCaseImpl` удаляется (вместе с тестом-времянкой фазы 1).

## Тест-зоны

- **W1 (androidTest, data)**: удаление значения опоры → потомки soft-deleted; смена CHOICE-значения → сброс только поддерева старой опции; мульти-цель гаснет последним; удаление последнего значения ядра НЕ удаляет лексему.
- **W2 (unit, app)**: UseCase — деградация в черновик вместо LexemeCascadeRemoved; выдача типов с опциями; фильтр снят.
- **W3 (unit, wordcard reducer)**: addable-список по правилу участия (draft: только ядра; заполнил ядро → открылись зависимые от лексемы; выбрал опцию → открылся value-зависимый); CHOICE-контент state (выбор, смена, commit); full-state assert.
- **W4**: регрессия всех модулей + сборка + линт + девайс-чек-лист (объединённый: фаза 1 + фаза 2 критерии).
