# IS485 — Редизайн интерфейса приложения Lexeme

Тикет: [#485](https://github.com/KilgoreT/PolyTrainer/issues/485).

## Задача

Спроектировать новый визуальный дизайн Android-приложения **Lexeme** (изучение иностранных слов: личные словари + карточки слов + квизы). Функциональность и логика экранов НЕ меняются — меняется облик: палитра, типографика, компоновка, стиль карточек/чипов/кнопок, empty-state'ы, состояния ошибок.

Материал в этой папке самодостаточен: скриншоты текущего приложения + по документу на каждый экран с полным описанием интерфейса (каждый элемент: смысл, точные строки RU/EN с ресурсными ключами, доменная сущность, значимость, поведение и состояния). Всё сверено с кодом приложения.

## Экраны без скриншотов (дизайнить в той же системе)

- **Менеджер компонентов** (Настройки → Компоненты): как экран 04, но для всех компонентов. Уникальное: блок «Охват» — radio «Общий / Пословарный»; при «Пословарном» — multi-select чипы словарей (один диалог создаёт N компонентов); строки группируются по (название + тип + «Несколько») с суммарным счётчиком и чипами словарей. Спека: `docs/handbook/specs/component-constructor/spec.md`.
- **Все словари** (Настройки → Все словари): список словарей; тап = редактирование (экран 01 в режиме edit); корзина = диалог каскадного удаления («все слова, определения и результаты квизов будут удалены»); при удалении текущего словаря текущим становится первый оставшийся; пустое состояние «Нет словарей / Создайте первый!» (топбар скрыт). Спека: `docs/handbook/specs/dictionary-list/spec.md`.
- **Квизы** (таб + чат квиза) и **Статистика** (таб) — отдельные табы, вне скоупа скриншотов, но нижняя навигация — общая.
- **Конфиденциальность** (WebView) и **О приложении** — описаны в [05_settings.md](05_settings.md).
- Диалоги подтверждения, bottom sheet добавления слова, снекбары — описаны в доках своих экранов (Undo — только на карточке слова).

## Доменная модель (общая для всех экранов)

- **Словарь (Dictionary)** — контейнер слов; привязан к языку/стране (флаг). Юзер держит несколько словарей и переключается между ними.
- **Слово (Word)** — запись в словаре; строка на изучаемом языке.
- **Значение / Лексема (Lexeme)** — одно значение слова; у слова их может быть несколько (book → «книга», «бронировать»). Состоит из значений компонентов.
- **Компонент (ComponentType)** — тип поля лексемы. Built-in: «Перевод» (есть всегда). Пользовательские: название, тип значения (Текст/Изображение), «Несколько» (isMultiple), охват (Общий / Пословарный).
- **Значение компонента (ComponentValue)** — конкретное содержимое (текст перевода, пример…) внутри лексемы.

## Терминология (фиксирована, менять нельзя)

Охват = scope · Общий = global · Пословарный = per-dictionary · Несколько = multiple. Для лексемы юзерский термин — «значение»/«meaning» (в текущих строках расщеплён с «value» — при редизайне выровнять, см. [03_word_card.md](03_word_card.md)).

## Шкала значимости элементов (используется в документах)

- **primary** — ядро сценария экрана; без него экран теряет смысл.
- **secondary** — поддерживающий элемент; важен, но не главный фокус.
- **service** — служебный/навигационный; доступен, но не привлекает внимание.

## Известные заглушки текущего UI (помечены ⚠️ в доках)

Дизайнить нужно ЦЕЛЕВОЕ поведение, а не заглушку: «Переместить» в режиме выделения (no-op, пикер словаря не существует), флаг в шапке карточки слова (захардкожен UK), прогресс/результат экспорта-импорта (красная плашка, результат не показывается), «Показать все (N)» в блоке downgrade (no-op), непереведённые хардкод-строки («Слово удалено», «Press + to add word», снеки компонентов на EN).
