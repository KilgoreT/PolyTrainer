# 03 — Карточка слова

Скриншоты: [03_word_card_v1.png](03_word_card_v1.png) (слово без значений), [03_word_card_v2.png](03_word_card_v2.png) (пустой черновик значения), [03_word_card_v3.png](03_word_card_v3.png) (ввод перевода, клавиатура), [03_word_card_v4.png](03_word_card_v4.png) (значение заполнено).

## Назначение

Детальный экран слова: шапка слова + список его значений (лексем). Здесь создаются/редактируются/удаляются значения и их компоненты. Экран только что переработан в IS481 под generic-модель компонентов — редизайн обязан сохранить эту механику (чипы доступных компонентов, инлайн-редактирование).

**Доменные сущности экрана:** Word (шапка), Lexeme (карточки значений), ComponentType (чипы добавления), ComponentValue (поля значений).

## Структура

Топбар (назад + меню) → карточка-шапка слова → карточки значений (0..N) → FAB «добавить значение» (низ-право). Фон — светлый оттенок брендового цвета (лавандовый), карточки белые.

## Элементы

### Кнопка «назад» (топбар)

- **Смысл:** возврат к списку слов. При выходе экран сам дожимает незакоммиченные правки (flush-on-back с блокирующим лоадером).
- **Значимость:** service.

### Меню «⋮» (топбар)

- **Смысл:** overflow-меню; сейчас в нём ровно один пункт — «Удалить» слово.
- **Текст пункта:** RU «Удалить» / EN «Delete» (`button_delete`), красный destructive-стиль с иконкой корзины.
- **Значимость:** service.
- **Поведение:** пункт открывает диалог подтверждения удаления слова. ⚠️ Меню из одного пункта — при редизайне решить: либо прямая иконка удаления, либо меню наполняется (share, перенос в другой словарь и т.п.).

### Карточка-шапка слова

- **Смысл:** само слово крупно (`book`), дата добавления, флаг словаря.
- **Текст:** лейбл даты — RU «Добавлено» / EN «Added» (`word_card_added_field`) + дата в формате «d MMMM yyyy» («3 июля 2026»); слово — контент юзера.
- **Доменная сущность:** Word (+ принадлежность Dictionary — флаг).
- **Значимость:** primary (контекст всего экрана).
- **Поведение:** слово редактируется ИНЛАЙН — тап по слову открывает режим редактирования прямо в шапке, сохранение по потере фокуса; ввод блокируется во время фоновых операций и выхода с экрана. ⚠️ Флаг в шапке сейчас захардкожен заглушкой (британский флаг всегда, независимо от словаря — на скриншотах словарь US, флаг UK) — в редизайне флаг обязан быть реальным флагом словаря.

### Карточка значения (лексемы)

- **Смысл:** одно значение слова. Содержит: значения компонентов (заполненные поля), чипы доступных к добавлению компонентов, кнопку удаления значения.
- **Доменная сущность:** Lexeme + её ComponentValues.
- **Значимость:** primary (ядро экрана).
- **Состояния (по скриншотам):**
  - **Пустой черновик** (v2): карточка без контента, справа чип «Перевод +», внизу «Удалить». Черновик ещё не в БД — реально создастся при вводе первого значения.
  - **Редактирование** (v3): после тапа по чипу открывается инлайн-поле: лейбл-чип «Перевод ×» (брендовый) + текстовое поле с курсором и системной клавиатурой. Сохранение — по уходу с поля (пустое значение отбрасывается).
  - **Заполнено** (v4): лейбл-чип «Перевод ×» + текст значения («книга») обычным текстом; «Удалить» внизу.

- **Правило «один активный редактор»:** одновременно редактируется только одно поле. Тап по другому чипу/полю или по фону экрана авто-коммитит текущее открытое поле (пустое — исчезает). Фокус-модель определяет переходы/анимации полей.
- **⚠️ «×» на лейбл-чипе — деструктивное действие:** у СОХРАНЁННОГО значения «×» удаляет значение сразу, БЕЗ диалога подтверждения. Очистить текст и увести фокус — тоже удаление. Удаление последнего значения каскадно удаляет всю лексему (со снеком Undo). При редизайне решить, оставлять ли столь лёгкое удаление без подтверждения.
- **Image-компоненты чипов НЕ дают:** ряд чипов фильтрует только текстовые шаблоны; компонент с шаблоном «Изображение» на карточке не появляется — рисовать image-чип не нужно.

### Чип добавления компонента («Перевод +»)

- **Смысл:** каждый доступный в словаре компонент показан чипом; тап добавляет пустое поле этого компонента в значение. Built-in «Перевод» есть всегда; пользовательские компоненты словаря добавляют свои чипы. Компонент без «Несколько» после заполнения из чипов исчезает (одно значение на карточку), с «Несколько» — остаётся.
- **Текст:** RU «Перевод» / EN «Translation» (`word_card_bottom_translation`); имена пользовательских компонентов — контент юзера.
- **Доменная сущность:** ComponentType.
- **Значимость:** primary (механика наполнения карточки).

### Кнопка «Удалить» (в карточке значения)

- **Смысл:** удалить значение целиком.
- **Текст:** RU «Удалить» / EN «Remove» (`word_card_lexeme_remove`); иконка — корзина; красный цвет (destructive).
- **Доменная сущность:** удаление Lexeme.
- **Значимость:** secondary (должна быть доступна, но не провоцировать).
- **Поведение:** диалог подтверждения (RU «Удалить значение?» / `word_card_delete_lexeme`) → удаление → снек с Undo (RU «Отменить» / `word_card_snackbar_undo`). Диалог обязан закрываться по подтверждению (IS481 BUG-2).

### FAB «добавить значение»

- **Смысл:** создать новое значение (черновик лексемы).
- **Текст:** нет (иконка «документ+»); accessibility — RU «Добавить значение» / EN «Add meaning» (`word_card_add_lexeme`).
- **Доменная сущность:** создание Lexeme (черновик).
- **Значимость:** primary.
- **Поведение:** тап → появляется пустой черновик (v2). Пока черновик уже открыт — FAB приглушён/неактивен (v2: бледный).

## Состояния экрана

1. Слово без значений (v1) — только шапка + FAB.
2. Пустой черновик значения (v2) — FAB приглушён.
3. Инлайн-ввод значения с клавиатурой (v3).
4. Значение заполнено (v4).

Служебные/ошибочные состояния (в скриншоты не попали — дизайнить обязательно):

5. **Начальная загрузка** — сейчас пустой экран БЕЗ спиннера (⚠️ дыра текущего UI; в редизайне нужен loading-state).
6. **Фоновая операция** (`isPendingDbOp`) — все интерактивы экрана заблокированы до ответа БД; видимой индикации сейчас нет.
7. **Выход с несохранёнными правками** — блокирующий лоадер-оверлей, пока правки дожимаются в БД (flush-on-back).
8. **Ошибки** — снекбары: «не удалось загрузить компоненты» с кнопкой «Повторить»; ошибка сохранения значения — поле остаётся В РЕДАКТИРОВАНИИ с введённым текстом + ошибка-снек; ошибка восстановления (Undo) — снек с «Повторить»; «слово не найдено» — молчаливый выход с экрана.

## Терминология UI (⚠️ расщеплена)

Лексема в строках называется по-разному: FAB — «Add meaning»/«Добавить значение», диалоги и снеки — «value»/«значение» («Delete value?», «Value deleted»). Для редизайна зафиксировать ЕДИНЫЙ термин (рекомендация: «значение»/«meaning» — выбрать одну пару и выровнять все строки).

## Заметки для дизайна

- Механика «чипы компонентов → инлайн-поля» — свежая (IS481) и фиксированная; редизайн меняет облик, не поведение.
- У слова без значений (v1) нет подсказки, что делать дальше — рассмотреть empty state внутри экрана.
- Терминология: «значение» (для юзера) = Lexeme; «Перевод» — built-in компонент; не вводить в дизайн слово «лексема».
- Учесть длинные значения и несколько компонентов на карточке (скриншоты показывают только перевод; пользовательские компоненты добавляют чипы и поля в те же карточки).
