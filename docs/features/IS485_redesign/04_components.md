# 04 — Компоненты словаря

Скриншоты: [04_components_v1.png](04_components_v1.png) (пустое состояние), [04_components_v2.png](04_components_v2.png) (диалог создания, пустой), [04_components_v3.png](04_components_v3.png) (диалог заполнен), [04_components_v4.png](04_components_v4.png) (список компонентов).

## Назначение

Конструктор компонентов конкретного словаря: здесь юзер создаёт свои поля для карточек слов (примеры, определения, заметки…) и управляет ими. Открывается из словаря; заголовок — имя словаря. Есть родственный экран «Компоненты» из Настроек (менеджер всех компонентов, в скриншоты не попал) — дизайн должен масштабироваться на оба.

**Доменная сущность экрана:** ComponentType (пословарные + видимые общие) — CRUD.

## Структура

Топбар (назад + имя словаря) → контент (empty state ИЛИ список компонентов) → FAB «+ Создать» (низ-право). Диалог создания — модальный, поверх затемнения.

## Элементы

### Топбар: «назад» + заголовок

- **Смысл:** заголовок = название словаря («Английский») — контекст, чьи компоненты редактируем.
- **Текст:** динамический (имя словаря). Связанный ключ: RU «Компоненты словаря» / EN «Components in dictionary» (`per_dict_components_title`).
- **Значимость:** service.

### Пустое состояние (v1)

- **Смысл:** в словаре нет пользовательских компонентов — есть только built-in перевод. Объясняет, зачем нужны компоненты, и ведёт к созданию.
- **Текст:**
  - иконка: пиктограмма «квадраты + плюс» (символ компонентов);
  - заголовок: RU «В этом словаре только перевод» / EN «Only translation in this dictionary» (`components_empty_headline_per_dict`);
  - тело: RU «Добавьте новый компонент, чтобы расширить карточки.» / EN «Add another component to extend your cards.» (`components_empty_body_per_dict`);
  - CTA-кнопка: RU «Создать компонент» / EN «Create component» (`components_empty_cta`).
- **Доменная сущность:** ComponentType (отсутствие пользовательских).
- **Значимость:** primary (для нового юзера — главный проводник в фичу).
- **Поведение:** CTA открывает диалог создания. ⚠️ Дублируется с FAB «+ Создать» на том же экране — при редизайне решить, оставлять ли оба.

### FAB «+ Создать»

- **Смысл:** создать компонент (то же действие, что CTA пустого состояния; единственный вход при непустом списке).
- **Текст:** RU «Создать» / EN «Create» (`components_create_cta`), расширенный FAB с иконкой «+».
- **Значимость:** primary.

### Диалог «Новый компонент» (v2, v3)

- **Смысл:** форма создания компонента.
- **Элементы:**
  - Заголовок: RU «Новый компонент» / EN «New component» (`components_create_dialog_title`).
  - Поле «Название»: RU «Название» / EN «Name» (`components_create_field_name`) — имя компонента (v3: «Пример»); контент юзера. Уникальность в охвате проверяется (ошибки «уже есть в этом/другом охвате»).
  - Секция «Тип значения»: RU «Тип значения» / EN «Type of value» (`components_create_field_template`) — радио-выбор шаблона: RU «Текст» / EN «Text» (`components_template_text`) и RU «Изображение» / EN «Image» (`components_template_image`). Текст — единственный реально работающий сейчас шаблон.
  - Чекбокс: RU «Разрешить несколько значений на карточке» / EN «Allow multiple values per card» (`components_create_field_is_multi`) — свойство «Несколько» (isMultiple).
  - Кнопки: RU «Отменить» / EN «Cancel» (`button_cancel`) и RU «Создать» / EN «Create» (`components_button_create`).
- **Доменная сущность:** ComponentType (name, template, isMultiple; охват — пословарный, т.к. создаётся из словаря).
- **Значимость:** primary.
- **Поведение:** «Создать» disabled при пустом имени (v2 — серая, v3 — активная). Редактирование существующего компонента — тот же диалог (поля `components_edit_*`), но с двумя guard'ами:
  - **смена шаблона запрещена** — radio «Тип значения» кликабелен, но submit отбивается снекбаром (⚠️ выглядит как разрешённая правка, падающая в конце — в редизайне лучше блокировать radio сразу);
  - **downgrade «Несколько» → одно значение блокируется**, если у компонента уже есть карточки с несколькими значениями: в диалоге появляется error-контейнер с превью затронутых карточек (топ-3) и кнопкой «Показать все (N)» (⚠️ кнопка сейчас no-op). Это отдельное пятое состояние диалога — дизайнить.

### Строка компонента в списке (v4)

- **Смысл:** один компонент с его свойствами и действиями.
- **Состав:**
  - Название («Пример») — контент юзера;
  - Бейдж «Несколько»: RU «Несколько» / EN «multiple» (`components_chip_multi`) — только если isMultiple;
  - Бейдж охвата: имя словаря («Английский») — пословарный компонент; у общих — RU «Общий» / EN «global»;
  - Иконка шаблона + бейдж «Текст»: RU «Текст» / EN «Text» (`components_template_text`);
  - Счётчик использования: RU «Значений: 0» / EN «%d values» (`per_dict_row_value_count`) — сколько значений этого компонента уже создано в карточках;
  - Действия: карандаш (редактировать) и корзина (удалить).
- **Доменная сущность:** ComponentType + агрегат использования (счётчик ComponentValue).
- **Значимость:** primary.
- **Поведение:** удаление — диалог подтверждения с превью последствий (в скриншоты не попал, дизайнить): заголовок RU «Удалить \"%1$s\"?» (`components_delete_dialog_title`) + динамические строки impact — «Будет скрыто значений: N» (`components_delete_impact_values`), «Затронуто словарей: N» (`components_delete_impact_dicts`), «Будет удалён из настроек квиза: N» (`components_delete_impact_quiz`), «Сбросит выбор квиза в словарях: N» (`components_delete_impact_prefs`), fallback «Превью последствий недоступно» (`components_delete_impact_unavailable`) + пояснение «Компонент будет помечен как удалённый. Значения остаются в базе, но скрываются.» (`components_delete_hint`). Это soft-delete, не каскад.

## Состояния экрана

1. Пустое состояние (v1).
2. Диалог создания, пустой (v2) — «Создать» disabled.
3. Диалог создания, заполнен (v3) — «Создать» enabled, чекбокс отмечен.
4. Список компонентов (v4).

Не попавшие в скриншоты (дизайнить):

5. **Загрузка** — спиннер.
6. **Ошибка первичной загрузки** — виджет «Не удалось загрузить…» + кнопка «Повторить».
7. **Диалог редактирования** с guard-состояниями (см. выше): запрет смены шаблона, блок downgrade «Несколько» с превью затронутых карточек.
8. **Снекбары результата операций** — ⚠️ сейчас захардкожены по-английски («Created N», «N values hidden», «Failed…») и БЕЗ Undo (Undo в приложении есть только на карточке слова). Для редизайна: локализуемые success/error-снеки; решить, нужен ли здесь Undo.

## Заметки для дизайна

- Строка компонента перегружена бейджами (4 бейджа + счётчик + 2 иконки на одну строку) — главный кандидат на визуальное упорядочивание.
- Терминология фиксирована (IS481): Охват/Общий/Пословарный/Несколько — RU; scope/global/per-dictionary/multiple — EN. Не переименовывать в дизайне.
- «Изображение» как шаблон в UI выбирается, но функционально не поддержан до конца — уточнить, показывать ли его в редизайне (риск мёртвой опции).
- Дизайн должен переноситься на «Менеджер компонентов» (Настройки → Компоненты): там те же строки, но с выбором охвата (Общий/Пословарный) и списком словарей.
