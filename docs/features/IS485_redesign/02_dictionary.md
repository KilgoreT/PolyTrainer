# 02 — Словарь (главный экран, таб)

Скриншоты: [02_dictionary_v1.png](02_dictionary_v1.png) (пустой словарь), [02_dictionary_v2.png](02_dictionary_v2.png) (список слов), [02_dictionary_v3.png](02_dictionary_v3.png) (слово с превью значения), [02_dictionary_v4.png](02_dictionary_v4.png) (открыт дропдаун выбора словаря).

## Назначение

Главный экран приложения: список слов текущего словаря. Отсюда начинаются все ключевые сценарии — добавить слово, открыть карточку слова, переключить словарь, уйти в тренировку.

**Доменные сущности экрана:** Dictionary (текущий словарь), Word (список), превью Lexeme/ComponentValue (чип под словом).

## Структура

Топбар (заголовок + 2 действия) → список карточек-слов → FAB «+» (низ-право) → нижняя навигация (4 таба).

## Элементы

### Заголовок «Словарь»

- **Смысл:** название таба.
- **Текст:** RU «Словарь» / EN «Dictionary» (`item_title_vocabulary`).
- **Значимость:** service.

### Иконка компонентов словаря (топбар)

- **Смысл:** вход в «Компоненты словаря» (экран 04) для текущего словаря. Визуально похожа на гантель, но это пиктограмма инструментов/компонентов.
- **Текст:** нет (иконка).
- **Доменная сущность:** ComponentType (конструктор компонентов Dictionary).
- **Значимость:** secondary.
- **Поведение:** тап → экран 04 с текущим словарём. Показывается только когда словарь выбран; во время загрузки словарей вместо действий топбара — спиннер. ⚠️ Иконка неочевидна (читается как «тренировка») — при редизайне подобрать однозначную метафору компонентов.

### Флаг текущего словаря (топбар)

- **Смысл:** индикатор текущего словаря и вход в его смену.
- **Текст:** нет (круглый флаг страны словаря).
- **Доменная сущность:** Dictionary (текущий).
- **Значимость:** primary (единственный индикатор «в каком я словаре»).
- **Поведение:** тап → дропдаун (v4): список словарей (флаг + название, галочка у текущего — «Английский ✓») + пункт «Добавить словарь» (RU `menu_item_title_add_dict` «Добавить словарь» / EN «Add dictionary», иконка «+в круге») → экран 01.

### Карточка слова (элемент списка)

- **Смысл:** одно слово словаря. Крупно — само слово (`plate`, `book`); ниже — ВСЕ его значения: на каждое значение чип перевода (v3: «книга», светло-фиолетовый чип) и, если есть, текст определения; между значениями — разделитель.
- **Текст:** контент юзера (динамический).
- **Доменная сущность:** Word + его Lexemes (перевод/определение каждой).
- **Значимость:** primary (ядро экрана).
- **Поведение:** тап → карточка слова (экран 03); длинный тап → режим выделения (см. «Состояния»). Слово без значений — карточка без чипов (v2: `plate`). ⚠️ Превью в списке пока показывает только перевод и определение (легаси-модель) — пользовательские компоненты из IS481 в превью не выводятся; решить в редизайне, что показывать.

### Пустое состояние (v1)

- **Смысл:** словарь без слов; сейчас — едва различимая «водяная» иллюстрация на фоне, без текста и CTA.
- **Значимость:** secondary.
- **Поведение:** ⚠️ нет подсказки «добавь первое слово» — при редизайне сделать полноценный empty state (иллюстрация + текст + указание на FAB).

### FAB «+»

- **Смысл:** добавить слово в текущий словарь.
- **Текст:** нет (иконка «+», брендовый цвет, скруглённый квадрат).
- **Доменная сущность:** создание Word.
- **Значимость:** primary (главное действие экрана).
- **Поведение:** тап → bottom sheet добавления слова (см. ниже); FAB анимированно скрывается, пока шит открыт.

### Bottom sheet добавления слова (в скриншоты не попал — дизайнить обязательно)

- **Смысл:** главный сценарий приложения — добавить слово. Модальный bottom sheet: одно текстовое поле + кнопка подтверждения (активна при непустом вводе).
- **Скрытая функция:** ввод в поле live-фильтрует список слов позади шита — работает как ПОИСК по словарю (юзер видит, есть ли уже такое слово). Другого поиска на экране нет — при редизайне решить, оставлять поиск скрытым в шите или выносить.
- **Повторное использование:** этот же шит открывается для редактирования слова из режима выделения (с префиллом).
- **Доменная сущность:** Word (создание/редактирование) + фильтр списка.
- **Значимость:** primary.

### Нижняя навигация (4 таба)

- **Смысл:** корневая навигация приложения. Присутствует на всех табовых экранах (02, Квизы, Статистика, 05).
- **Текст:** RU «Словарь / Квизы / Статистика / Настройки» / EN «Dictionary / Quizzes / Statistics / Settings» (`item_title_vocabulary`, `item_title_training`, `item_title_stats`, `item_title_settings`).
- **Иконки:** блокнот / шапка-«диалог» / график / шестерёнка.
- **Значимость:** service (но постоянно на виду — стиль критичен).
- **Поведение:** активный таб подсвечен брендовым цветом (иконка + подпись).

## Состояния экрана

1. **Пустой словарь** (v1) — см. «Пустое состояние».
2. **Список слов** (v2) — слова без значений.
3. **Слово со значением** (v3) — чип-превью перевода под словом.
4. **Дропдаун словарей** (v4) — поверх контента, якорь — флаг в топбаре.
5. **Режим выделения** (в скриншоты не попал, есть в приложении): длинный тап по слову → топбар меняется на контекстный (тёмный): «×» (выход из режима), счётчик выделенных, действия — редактировать (только при одном выделенном; открывает тот же bottom sheet с префиллом), переместить, удалить (с диалогом подтверждения; после — снек «Слово удалено», ⚠️ хардкод RU, без Undo). Системная «назад» выходит из режима выделения, не с экрана. ⚠️ «Переместить» сейчас — no-op (пустой обработчик, пикера целевого словаря не существует): для редизайна это проектирование С НУЛЯ (включая пикер словаря), не рестайлинг.
6. **Загрузка** — по центру спиннер; в топбаре вместо действий — маленький спиннер.
7. **Нет ни одного словаря** (отличается от «словарь пуст», v1): тело — заглушка «Press + to add word» (⚠️ хардкод EN), флаг в топбаре скрыт, дропдаун содержит только «Добавить словарь». Два разных empty-state — нужны две разные подсказки.

## Технические факты для дизайна

- Список слов — на пагинации (Paging 3): дизайн должен переживать дозагрузку страниц.
- Обрезки длинных строк в карточке нет — карточка растягивается по контенту; переполнение (длинное слово/перевод/много значений) — задача редизайна.

## Заметки для дизайна

- Карточка слова сейчас информационно бедная: слово + один чип. Решить, что показывать (кол-во значений, прогресс изучения, дата) — но не перегружать.
- Дропдаун словарей — ключевой систематический паттерн (переключение контекста всего приложения); в редизайне может стать полноценным пикером/шитом.
- FAB и нижняя навигация конфликтуют за нижнюю зону — учесть отступы списка (последняя карточка не должна прятаться под FAB).
