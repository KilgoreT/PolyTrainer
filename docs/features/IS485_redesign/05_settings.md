# 05 — Настройки (таб)

Скриншот: [05_settings.png](05_settings.png).

## Назначение

Корневой экран настроек: точки входа в управление словарями и компонентами, экспорт/импорт данных, юридическая информация. Один из четырёх табов нижней навигации.

**Доменные сущности экрана:** Dictionary (управление), ComponentType (менеджер), данные приложения (экспорт/импорт).

## Структура

Заголовок «Настройки» → две группы-карточки пунктов (данные / информация) → нижняя навигация (активен таб «Настройки»).

## Элементы

### Заголовок

- **Текст:** RU «Настройки» / EN «Settings» (`item_title_settings`).
- **Значимость:** service.

### Группа 1 — данные

| Пункт | Текст RU / EN (ключ) | Иконка | Смысл / куда ведёт | Сущность | Значимость |
|---|---|---|---|---|---|
| Все словари | «Все словари» / «Dictionaries» (`settings_section_lang_management`) | глобус | Управление словарями: список, удаление; шеврон «>» — есть вложенный экран | Dictionary | primary |
| Компоненты | «Компоненты» / «Components» (`settings_section_components_management`) | пиктограмма компонентов | Менеджер ВСЕХ компонентов (общие + пословарные всех словарей, выбор охвата); шеврон «>» | ComponentType | primary |
| Экспорт данных в файл | «Экспорт данных в файл» / «Export data to file» (`settings_section_export`) | стрелка вверх | Выгрузка данных юзера в файл (бэкап); действие без вложенного экрана | все данные | secondary |
| Импорт данных | «Импорт данных» / «Import data» (`settings_section_import`) | стрелка вниз | Загрузка данных из файла; действие | все данные | secondary |

**Механика экспорта/импорта (для дизайна flow):** экспорт = системный SAF-пикер ПАПКИ → создание файла → системный share-диалог готового файла; импорт = системный пикер файла. ⚠️ Текущий «прогресс» — красная заглушка-плашка (Box 200dp), а результат операции (успех/ошибка) юзеру не показывается вообще (снек в коде есть, но не смонтирован). Для редизайна проектировать полный flow: запуск → прогресс → результат (успех со share / ошибка с повтором).

### Группа 2 — информация

| Пункт | Текст RU / EN (ключ) | Иконка | Смысл | Значимость |
|---|---|---|---|---|
| Конфиденциальность | «Конфиденциальность» / «Privacy Policy» (`settings_section_privacy_policy`) | щит | ВНУТРЕННИЙ WebView-экран политики (не внешняя ссылка) | service |
| О приложении | «О приложении» / «About the App» (`settings_section_about_app`) | «i» в круге | Экран «О приложении»; шеврон «>» | service |

**Экран «Конфиденциальность» (WebView, дизайнить состояния):** LinearProgress сверху при загрузке; экран ошибки «Не удалось загрузить страницу» + «Повторить»; переходы на внешние домены уходят в системный браузер.

**Экран «О приложении»:** логотип, название «Lexeme», версия «v{X}», текст-описание приложения (`settings_section_about_app_title`).

### Нижняя навигация

Та же, что на экране 02 (см. [02_dictionary.md](02_dictionary.md)); активен таб «Настройки».

## Заметки для дизайна

- Визуальная группировка (две карточки) — правильная; при редизайне сохранить смысловое деление «данные» / «информация».
- Непоследовательность шевронов: «Все словари», «Компоненты», «О приложении» имеют «>», «Экспорт»/«Импорт»/«Конфиденциальность» — нет. Осознанно (действие vs вложенный экран), но визуально выглядит случайно — унифицировать подачу.
- Экспорт/импорт — потенциально долгие операции: предусмотреть в дизайне состояние прогресса и результата (сейчас его не видно).
