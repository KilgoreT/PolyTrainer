# 01 — Создание словаря

Скриншоты: [01_create_dictionary.png](01_create_dictionary.png) (пустая форма), [01_create_dictionary_v2.png](01_create_dictionary_v2.png) (заполнена: выбран флаг, введено название, кнопка активна).

## Назначение

Создание нового словаря: юзер выбирает страну/язык (флаг) и даёт словарю название. Это первый содержательный экран для нового пользователя (без словарей приложение бесполезно) и обычный экран из меню «Добавить словарь» для существующего.

**Доменная сущность экрана:** Словарь (Dictionary) — создание.

## Структура

Сверху вниз: строка «выбранный флаг + поле названия» → поле поиска по флагам → прокручиваемая сетка флагов (5 колонок) → закреплённая внизу кнопка «Создать».

## Элементы

### Выбранный флаг (превью)

- **Смысл:** круглый слот слева от поля названия; показывает выбранный из сетки флаг.
- **Текст:** нет. До выбора флага плейсхолдер показывает ПЕРВУЮ БУКВУ вводимого названия (обновляется live по мере ввода); при пустом имени — пустой круг.
- **Доменная сущность:** Dictionary.numericCode (страна).
- **Значимость:** primary.
- **Поведение:** заполняется тапом по флагу в сетке (v2: флаг США); повторный тап по тому же флагу снимает выбор. Флаг опционален — словарь можно создать и без него.

### Поле «Название словаря»

- **Смысл:** имя словаря, которое юзер увидит в списках и дропдаунах. Вводится вручную; автоподстановки от выбранного флага нет (v2: «Английский» набран юзером).
- **Текст:** hint — RU «Название словаря» / EN «Dictionary name» (`dictionary_name_hint`).
- **Доменная сущность:** Dictionary.name.
- **Значимость:** primary.
- **Поведение:** обычный однострочный ввод; единственное обязательное поле — «Создать» активируется от непустого имени.

### Поле поиска по флагам

- **Смысл:** фильтрация сетки флагов по названию страны или языка.
- **Текст:** hint — RU «Поиск по стране или языку…» / EN «Filter flags…» (`dictionary_filter_flags_hint`). ⚠️ Локали рассинхронены по смыслу: RU обещает поиск по языку, EN — только «фильтр флагов». При редизайне выровнять.
- **Доменная сущность:** — (фильтр UI).
- **Значимость:** secondary.
- **Поведение:** иконка-лупа слева; фильтрует сетку по мере ввода.

### Сетка флагов

- **Смысл:** выбор страны/языка нового словаря. Круглые флаги, под каждым — название страны (длинные обрезаются: «Афгани…», «Америк…»).
- **Текст:** названия стран — из справочника стран (динамические, локализованные), не отдельные строковые ключи.
- **Доменная сущность:** справочник стран/языков → Dictionary.langCode.
- **Значимость:** primary (главная зона выбора).
- **Поведение:** вертикальный скролл; тап выбирает флаг (подсветка выбранного + перенос в превью). Ошибка дубля: RU «Словарь уже добавлен» / EN «Dictionary already added» (`dictionary_selection_error`).

### Кнопка «Создать»

- **Смысл:** подтверждение создания словаря.
- **Текст:** RU «Создать» / EN «Create» (`dictionary_selection_button`).
- **Доменная сущность:** команда создания Dictionary.
- **Значимость:** primary (главное действие).
- **Поведение:** два состояния — disabled (серая; v1) и enabled (брендовый цвет; v2). Активируется ТОЛЬКО непустым названием; выбор флага на активацию не влияет (флаг опционален). Закреплена у нижнего края. После создания — возврат с новым словарём. Экран также работает в режиме редактирования существующего словаря (форма префиллится именем и флагом).

## Режимы экрана (один лейаут — три контекста)

1. **Онбординг (setup)** — после splash, когда словарей нет: без топбара и без «назад» (выйти некуда), после сохранения → главный экран.
2. **Создание** — из дропдауна «Добавить словарь» или из «Все словари»: «назад» возвращает на предыдущий экран; созданный словарь становится текущим.
3. **Редактирование** — из «Все словари» (тап по словарю): форма префиллится именем и флагом, кнопка — «Сохранить» вместо «Создать».

## Состояния экрана

1. **Пустая форма** (v1): плейсхолдер-круг, пустое имя, «Создать» disabled.
2. **Заполнено** (v2): флаг выбран, имя введено (клавиатура открыта), «Создать» enabled.

## Заметки для дизайна

- Первый экран онбординга нового юзера — задаёт первое впечатление; сейчас выглядит как утилитарная форма.
- Онбординг-вариант и обычный вариант — фактически два разных лейаута одного экрана (с топбаром/без) — дизайнить оба.
- ⚠️ «Назад» теряет введённое молча, без подтверждения — решить, нужен ли confirm при заполненной форме.
- Обрезанные названия стран («Афгани…») — известная боль текущей сетки; при редизайне решить (размер ячейки, две строки, поповер).
