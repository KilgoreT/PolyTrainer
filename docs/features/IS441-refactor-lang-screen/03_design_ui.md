# IS441. Задача 3 — Дизайн UI

## Общая идея

Один экран, два состояния: **список словарей** и **форма создания/редактирования**. Переключение между ними внутри одного Scaffold — без модалок, без отдельных route.

---

## Состояние 1: Список словарей

```
┌──────────────────────────────┐
│ ← Словари                   │  AppBar (management) или без (setup)
├──────────────────────────────┤
│                              │
│  ┌────────────────────────┐  │
│  │ 🇬🇧 English        🗑️  │  │  элемент списка: флаг + название + иконка удаления
│  ├────────────────────────┤  │
│  │ 🇪🇸 Spanish        🗑️  │  │
│  ├────────────────────────┤  │
│  │ 📖 Биология        🗑️  │  │  словарь без флага → дефолтная иконка
│  └────────────────────────┘  │
│                              │
│                              │
│                              │
│   [ + Новый словарь ]        │  кнопка внизу → переключает на состояние 2
│                              │
└──────────────────────────────┘
```

**Тап на элемент** → переключение на состояние 2 (редактирование, форма заполнена данными словаря).

**Тап на 🗑️** → диалог подтверждения удаления.

**Пустое состояние** (нет словарей):

```
┌──────────────────────────────┐
│ ← Словари                   │
├──────────────────────────────┤
│                              │
│                              │
│       Нет словарей           │
│    Создайте первый!          │
│                              │
│   [ + Новый словарь ]        │
│                              │
└──────────────────────────────┘
```

## Состояние 2: Форма создания / редактирования

```
┌──────────────────────────────┐
│ ← Словари                   │  AppBar с кнопкой назад → возврат к списку
├──────────────────────────────┤
│                              │
│  Название                    │
│  ┌────────────────────────┐  │
│  │ English                │  │  текстовое поле, обязательно
│  └────────────────────────┘  │
│                              │
│  ☐ Привязать к языку         │  чекбокс, по умолчанию выключен
│                              │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─   │  ↓ появляется если чекбокс включён ↓
│                              │
│  Язык                        │
│  ┌──────────────────── ▼ ┐  │  выпадающий список с поиском
│  │ Испанский             │  │
│  └────────────────────────┘  │
│                              │
│  Флаг                        │
│  ┌────┬────┬────┬────┐       │
│  │ 🇪🇸 │ 🇲🇽 │ 🇦🇷 │ 🇨🇴 │       │  grid флагов стран языка
│  ├────┼────┼────┼────┤       │
│  │ 🇨🇱 │ 🇵🇪 │ 🇨🇺 │ 🇪🇨 │       │  выбранный выделен рамкой
│  └────┴────┴────┴────┘       │
│                              │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─   │
│                              │
│   [ Создать / Сохранить ]    │  активна когда название заполнено
│                              │
└──────────────────────────────┘
```

### Поведение элементов формы

**Название** — всегда видно, обязательно.

**Чекбокс "Привязать к языку":**
- Выключен → секции "Язык" и "Флаг" скрыты
- Включён → появляются "Язык" и "Флаг"

**Выбор языка (bottom sheet):**
- Тап на поле → открывается bottom sheet
- Сверху поле поиска
- Список всех языков (~180), фильтруется по вводу на любом языке
- Юзер вводит "исп" → фильтруется "Испанский"
- Тап на язык → bottom sheet закрывается, название языка в поле

**Флаги:**
- Появляются после выбора языка
- Если одна страна (японский → 🇯🇵) → автовыбор, показан один флаг уже выбранным
- Если несколько (испанский → 🇪🇸🇲🇽🇦🇷🇨🇴...) → горизонтальный скролл, тап для выбора

**Кнопка:**
- Создание: "Создать" — активна если название не пустое
- Редактирование: "Сохранить" — активна если что-то изменилось

## Переходы между состояниями

### Management режим

```
Список
  │
  ├── тап на элемент → Форма (редактирование, заполнена)
  ├── тап "+ Новый словарь" → Форма (создание, пустая)
  ├── тап 🗑️ → Диалог подтверждения → удаление → обновление списка
  │
Форма
  │
  ├── тап "← назад" → Список (без сохранения)
  ├── тап "Создать/Сохранить" → сохранение → Список (обновлённый)
```

### Setup (onboarding) режим

```
Форма (создание, сразу при открытии)
  │
  ├── тап "Создать" → Список (с одним созданным словарём + кнопка "Дальше")
  │
Список
  │
  ├── тап "Дальше" → закрытие экрана → главный flow
  ├── тап "+ Новый словарь" → Форма (создание ещё одного)
  ├── тап на элемент → Форма (редактирование)
```

Кнопка "Дальше" видна только в setup режиме. В management её нет.

## Диалог подтверждения удаления

Стандартный `AlertDialog`, не отдельный экран:

```
┌────────────────────────────┐
│  Удалить словарь?          │
│                            │
│  Все слова, определения    │
│  и результаты квизов       │
│  будут удалены.            │
│                            │
│  [ Отмена ]  [ Удалить ]   │
└────────────────────────────┘
```

---

## Решения

1. **Setup (onboarding)** — сразу форма создания. После создания → список с созданным словарём + кнопка "Дальше" → переход на главный flow.
2. **Кнопка "Новый словарь"** — внизу экрана, как сейчас (не FAB).
3. **Флаги** — grid (не горизонтальный скролл).
4. **Выбор языка** — bottom sheet с полем поиска (не DropdownMenu — 180 языков не влезут).
