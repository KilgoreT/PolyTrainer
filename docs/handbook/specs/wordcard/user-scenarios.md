# IS481 WordCard — Пользовательские сценарии

Формат каждого сценария: **Было** → **Шаги** → **Стало** → **🔧 Тех**.
- **Было / Шаги / Стало** — человеческим языком (что видит и делает юзер).
- **🔧 Тех** — мост к коду: `Msg` (что обрабатывается) · `Эффект` (что в БД) · `Стейт` (дельта значимых полей) · `Тесты` (эталон точных значений — в `07_test_design.md`).

Полные снимки стейта «все поля до→после» — в `07` (тест-контракт), здесь только дельта + ссылка.

Разделы: **L** — лексема · **V** — значения · **A** — загрузка/фон/ошибки.

## L2.8 — Создал черновик, ничего не ввёл, ушёл

**Было**
- Пустой черновик.

**Шаги**
1. Ухожу с экрана / начинаю редактировать слово.

**Стало**
- Пустой черновик исчез.
- «+ Лексема» снова доступна.

> Прим.: «создать другую лексему» в триггеры НЕ входит — пока висит черновик, `isCreatingLexeme=true` → «+ Лексема» серая, `CreateLexeme` = no-op (L1.3). Очистить пустой черновик созданием другой нельзя.

**🔧 Тех** (триггеры РАЗНЫЕ — не один механизм):
- **«Редактирую слово»** (`EnterWordEditMode`): Msg вызывает `commitAndCloseAllEdits` → пустой pristine дропнут + пустая `NOT_IN_DB` лексема удалена (B3), `canAddLexeme=true`. Тест: `CommitAndCloseAllEditsTest B3`.
- **«Ухожу с экрана»** (`NavigateBack`): сначала `commitAndCloseAllEdits` + `isExiting=true` (flush-on-back, 03 §6.2.3). Пустой черновик коммитить нечего → `hasInFlightCommits=false` → пост-шаг сразу эмитит `NavigationEffect.Back` (черновик при этом дропнут как пустой). Был бы несохранённый текст — досохранился бы под лоадером перед выходом (см. L7.1). Эффект: `NavigationEffect.Back`.

## L7.1 — Вышел с экрана с несохранённым → дожимается и выходит

**Было**
- Печатаю значение (поле открыто, текст не сохранён — фокус не убирал).

**Шаги**
1. Нажимаю «назад».

**Стало**
- Кратко появляется лоадер (идёт сохранение).
- Введённый текст сохраняется, затем экран закрывается.
- Если сохранение упало — экран НЕ закрывается, показывается ошибка, текст цел (можно повторить).

**🔧 Тех** (flush-on-back, маркеры — 03 §6.2.3)
- Msg: `NavigateBack` → `commitAndCloseAllEdits` + `isExiting=true`; навигацию даёт пост-шаг `reduce`, когда нет летящих записей (`hasInFlightCommits=false`)
- Эффект: `UpsertComponentValue` (commit) → успех `RefreshLexemeComponents` → `NavigationEffect.Back`; ошибка → `OperationFailed` (остаёмся)
- Стейт: `isExiting=true` (лоадер); компонент `isCommitting=true` пока пишется; успех → выход; ошибка → `isExiting=false`, поле цел
- Тесты: `NavigateBackTest` (§1.12)

# Решения по спорным вопросам (все закрыты)

1. **Rename типа в Manager (A12):** лейбл значения СЛЕДУЕТ за переименованием (резолв по `componentTypeId` из живого справочника, fallback на снимок — на окно до прихода `ComponentTypesLoaded`; значение с УДАЛЁННЫМ типом не грузится, см. A9). Связь в БД по id.
2. **Удаление несохранённого черновика (L3.6):** confirm-диалог для всех лексем, как сейчас; спец-обработки черновика нет.
3. **Ошибка update слова (term):** не трогаем — редактирование слова это IS479, вне скоупа.
4. **Ошибка restore/undo (A17):** retry-снек с тем же snapshot. Корень (soft-delete лексем/значений) → backlog `## ВекторныйПиздеж`.
5. **Undo при pending:** не реальный флоу (операции сериализованы `isPendingDbOp`, тычки последовательны) — спец-обработки нет, `UndoRestoreLexeme` остаётся не-guarded.
6. **Пусто vs пробелы (V10):** в фиче работают оба маршрута (покрыты тестами); унификация UI на `isBlank()` → backlog.
7. **Фидбэк single-draft guard (L1.3):** disabled через `canAddLexeme` (не прятать — мигание на transient pending), существующее поведение.
8. **Пустое значение (P1):** пустое удаляется **автоматически** (пустой pristine исчезает; очищенное сохранённое — удаляется) — как сейчас работает translation/definition. Brief §2 («пустое остаётся, юзер сам удалит trash») **отменён** этим решением.
9. **Multi-add (P2):** **один активный редактор**. Тап по другому +чипу делает commit-and-close текущего поля (пустое исчезает, с текстом — сохраняется), затем открывает новое. Двух одновременно открытых полей нет → стопки инпутов из brief §1 нет. Доп. блокировка +чипа НЕ вводится (single-active-editor уже разруливает; sub-second двойное создание на новой лексеме — backlog `ВекторныйПиздеж`). «Несколько одновременно открытых полей» — отдельная история в backlog.
10. **Удаление типа = каскад (A9):** удаление типа в Manager каскадно soft-удаляет ВСЕ его значения у всех лексем (`componentValueDao.softDeleteByTypeId`); маппер загрузки их фильтрует (`value.removedAt==null && type.removedAt==null`). На карточке значения удалённого типа не существует. **Orphan-модель (значение переживает тип) — отменена.** Android single-screen — гонки «карточка открыта, тип удалён параллельно» нет. Поэтому редактировать/удалять значение удалённого типа на карточке невозможно; `check(type.removedAt==null)` в `updateComponentValue` остаётся защитой data-слоя (в WordCard недостижима).

В backlog `## ВекторныйПиздеж` вынесено: soft-delete лексем/значений (корень A17); унификация пусто-vs-пробелы (V10).
