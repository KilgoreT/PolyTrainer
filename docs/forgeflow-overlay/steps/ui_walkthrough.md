Шаг Discovery ui sub-flow. Собрать факты о реальных UI-примитивах, существующих экранах-аналогах, теме и ресурсах проекта — релевантных будущей разметке фичи.

Цель — чтобы следующий шаг (`ui_layout`) писал layout **не теоретически, а на фактах**: какие виджеты уже есть в `modules/core/ui/*`, как устроены ближайшие экраны, какие токены темы используются.

## Что делать

1. Прочитай `scope_analysis.output` — пойми что за фича (домен UI, целевой экран/виджет).
2. Грепай реальный UI-код проекта. Какие именно сведения нужны — твоё решение. Кандидаты:
   - **UI-примитивы** — `modules/core/ui/*`: какие composable-обёртки есть (`PtButton`, `PtTextField`, контейнеры и т.п.). Сигнатуры и параметры.
   - **Экран-аналог** — composable ближайшего по домену экрана: структура layout'а, использованные виджеты, навигация, обработка состояний.
   - **Тема** — `theme/`: палитра цветов, типографика, отступы — что доступно через `MaterialTheme` или проектные wrapper'ы.
   - **Ресурсы** — строки, drawable, dimens которые могут переиспользоваться.
3. Если аналогичный экран / виджет существует — найди и сошлись на конкретные файлы и строки. Если не существует — зафиксируй явно «аналогов нет в проекте».
4. Запиши факты в `ui_walkthrough.md`.

## Принципы

- **Факты — не выводы.** Walkthrough не принимает дизайн-решений и не моделирует layout. Только что есть в коде сейчас.
- **Ссылки `file:line` обязательны.** Не пересказывай — цитируй или ссылайся на конкретное место.
- **Не дублируй гайды.** Правила построения layout / использования виджетов применяются на `ui_layout`, не здесь.
- **Если ничего подходящего нет — это тоже факт.** Не притягивай несуществующий аналог.

## Формат `ui_walkthrough.md`

Структура свободная — выбирай разделы по факту того что нашёл. В конце артефакта **обязательная** секция:

```markdown
## Вердикт

Аналог: <найден / не найден>
<краткое обоснование>
```

## Что НЕ делать

- Не писать код, не править файлы.
- Не принимать архитектурных решений.
- Не дублировать конвенции из гайдов проекта.
- Не задавать вопросов пользователю — если фактов мало, фиксируй что нашёл и переходи к вердикту.
